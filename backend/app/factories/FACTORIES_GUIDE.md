# Factories Usage Guide

## Overview

The `factories` module provides factory_boy factories for generating test data for all models in the Numo application.

## Available Factories

### CategoryFactory

Generates random Category instances with auto-generated names using Faker.

```python
from app.factories import CategoryFactory

# Build a single instance (not saved to DB)
category = CategoryFactory.build()
print(category.name)  # Random word

# Create and save to DB
category = CategoryFactory.create()  # Requires active DB session context

# Build multiple instances
categories = CategoryFactory.build_batch(5)
```

### PlaceFactory

Generates random Place instances with company names.

```python
from app.factories import PlaceFactory

# Build a single instance
place = PlaceFactory.build()
print(place.name)  # Random company name

# Create multiple
places = PlaceFactory.create_batch(3)
```

### DebitFactory

Generates Debit instances with related Category and Place instances.

```python
from app.factories import DebitFactory

# Build with auto-generated category and place
debit = DebitFactory.build()
print(debit.amount)  # Random amount between 0-999.99
print(debit.category.name)  # Random category
print(debit.place.name)  # Random place
print(debit.expensed_at)  # Random date

# Build multiple
debits = DebitFactory.build_batch(10)
```

### CreditFactory

Generates Credit instances with auto-generated timestamps and amounts.

```python
from app.factories import CreditFactory

# Build a single instance
credit = CreditFactory.build()
print(credit.amount)  # Random amount between 0-9999.99
print(credit.observations)  # Random sentence

# Create multiple
credits = CreditFactory.create_batch(5)
```

## Usage in Tests

Example test file using factories:

```python
import pytest
from app.factories import CategoryFactory, DebitFactory, PlaceFactory
from app.database import SessionLocal

@pytest.fixture
def sample_debits():
    \"\"\"Create test debits.\"\"\"
    return DebitFactory.build_batch(5)

    \"\"\"Test debit retrieval.\"\"\"
    assert len(sample_debits) == 5
    for debit in sample_debits:
        assert debit.amount > 0
        assert debit.category is not None
        assert debit.place is not None
```

## Factory Methods

### Build (Untracked)

- `Factory.build()` - Creates an instance in memory (not saved)
- `Factory.build_batch(n)` - Creates n instances

### Create (Database Save)

- `Factory.create()` - Creates and saves to database
- `Factory.create_batch(n)` - Creates and saves n instances

**Note**: When using `.create()`, ensure you're within an active database session context.

## Customization

You can override factory attributes to create specific instances:

```python
from app.factories import CategoryFactory, DebitFactory

# Create category with specific name
food_category = CategoryFactory.build(name='Food')
print(food_category.name)  # 'Food'

# Create debit with specific category
specific_debit = DebitFactory.build(category=food_category, amount=50.00)
print(specific_debit.amount)  # 50.0
```

## Generated Data Examples

- **Categories**: Random single words (auto, thought, anything)
- **Places**: Random company names (Lawson & Partners, Tech Solutions Inc)
- **Amounts**: Decimal values (debits up to 999.99, credits up to 9999.99)
- **Timestamps**: Random datetimes
